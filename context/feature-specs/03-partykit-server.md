# Feature 03 — PartyKit Signaling Server

Implement `party/server.ts` — the WebSocket relay that enables WebRTC handshake between two devices. This server does exactly one job: forward signaling messages between two peers while enforcing a hard 2-connection limit per room.

No business logic, no file content, no sessions stored. Only SDP and ICE messages pass through here.

---

## How the server fits into the session flow

```
Receiver opens /receive
  └─ generates code "ABCD12"
  └─ opens WebSocket to PartyKit room "fs-ABCD12"   ← first connection

Sender scans QR → opens /send?code=ABCD12
  └─ opens WebSocket to PartyKit room "fs-ABCD12"   ← second connection
  └─ server sends { type: "peer-joined" } to receiver

Receiver sends SDP offer → server relays to sender
Sender sends SDP answer  → server relays to receiver
Both exchange ICE candidates via server

WebRTC data channel opens (P2P, direct) — signaling no longer needed
Any third device attempts to join → server sends { type: "room-full" } and closes socket
```

---

## Message protocol

All messages are JSON strings. The server never parses content — it only inspects `type` for its own events.

### Client → Server → Client (relayed by server, opaque to server)

| type | Sent by | Description |
|------|---------|-------------|
| `offer` | Receiver | SDP offer, triggers after peer-joined |
| `answer` | Sender | SDP answer |
| `ice-candidate` | Either | ICE candidate |

### Server → Client (generated by server)

| type | Sent to | When |
|------|---------|------|
| `peer-joined` | First peer (receiver) | Second peer joins the room |
| `peer-left` | Remaining peer | Other peer disconnects |
| `room-full` | Third+ peer | Room already has 2 connections |

## Implementation — `party/server.ts`

Replace the current stub with:

```typescript
import type * as Party from "partykit/server"

export default class Server implements Party.Server {
  private peers = new Set<string>()

  constructor(readonly room: Party.Room) {}

  onConnect(conn: Party.Connection) {
    if (this.peers.size >= 2) {
      conn.send(JSON.stringify({ type: "room-full" }))
      conn.close()
      return
    }

    this.peers.add(conn.id)

    if (this.peers.size === 2) {
      // Second peer joined — tell the first peer to send its SDP offer
      this.room.broadcast(JSON.stringify({ type: "peer-joined" }), [conn.id])
    }
  }

  onMessage(message: string, sender: Party.Connection) {
    // Relay all signaling messages to the other peer
    this.room.broadcast(message, [sender.id])
  }

  onClose(conn: Party.Connection) {
    this.peers.delete(conn.id)
    this.room.broadcast(JSON.stringify({ type: "peer-left" }))
  }
}
```

### Why this works

- `this.peers` is a `Set<string>` of connection IDs, maintained in memory per room instance.
- `onConnect` checks the count **before** adding — if already 2, the new connection is rejected immediately.
- `onMessage` relays everything to everyone except the sender (`[sender.id]` is the exclusion list).
- `onClose` removes the peer and notifies the remaining one — receiver can reset its QR display.
- The server never inspects SDP or ICE content. It is transport-only.

## Local dev

```bash
npx partykit dev
```

PartyKit runs on `http://localhost:1999` by default. Update `.env.local`:

```
NEXT_PUBLIC_PARTYKIT_HOST=localhost:1999
```

---

## Deploy

```bash
npx partykit deploy
```

After deploy, PartyKit prints the live host URL (e.g. `flow-send.username.partykit.dev`). Update `.env.local` and Vercel env:

```
NEXT_PUBLIC_PARTYKIT_HOST=flow-send.username.partykit.dev
```

## Verification

Test with two browser tabs pointing at the same room name.

### Manual smoke test

```javascript
// Tab 1 (receiver — joins first)
const ws1 = new WebSocket("ws://localhost:1999/parties/main/fs-TEST01")
ws1.onmessage = e => console.log("Tab1 received:", e.data)

// Tab 2 (sender — joins second)
const ws2 = new WebSocket("ws://localhost:1999/parties/main/fs-TEST01")
ws2.onmessage = e => console.log("Tab2 received:", e.data)

// Tab 1 should immediately print: {"type":"peer-joined"}

// Tab 3 (should be rejected)
const ws3 = new WebSocket("ws://localhost:1999/parties/main/fs-TEST01")
ws3.onmessage = e => console.log("Tab3 received:", e.data)
// Tab 3 should print: {"type":"room-full"}

// Relay test
ws2.send(JSON.stringify({ type: "offer", sdp: "test-sdp" }))
// Tab 1 should print: {"type":"offer","sdp":"test-sdp"}

// Disconnect test
ws2.close()
// Tab 1 should print: {"type":"peer-left"}
```
